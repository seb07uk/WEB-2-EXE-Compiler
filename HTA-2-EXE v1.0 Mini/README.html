Step-by-step Internally

  1. All project files are collected â€” HTA, CSS, JS, JSON, XML, images, fonts and more
  2. External <script src="..."> and <link href="..."> references are inlined directly into the HTA source â€” no broken relative paths at runtime
  3. All files are base64-encoded and embedded as string literals in a Python wrapper script
  4. PyInstaller --onefile --noconsole compiles the wrapper into a single .exe
  5. Windows VERSIONINFO resource is injected via a version_info.txt spec file
  6. Optionally, UPX compresses the result for smaller distribution size

ğŸ“¦ UPX Compression â€” Deep Dive

After PyInstaller finishes, the converter runs:

--best selects maximum compression. --lzma uses the LZMA algorithm for better ratios than the default NRV2B. After compression, the log reports results:

[INFO] UPX: 8420 KB â†’ 3210 KB (saved 5210 KB, 38.1% of original)

Typical compression: 40â€“70% size reduction depending on content type.

UPX exit code 1 is treated as success (file already compressed or compression not beneficial â€” UPX does not modify the file in this case).

ğŸ“¦ Asset Bundling & Inlining

CSS & JavaScript Inlining

Before bundling, external references in the HTA source are detected and inlined:

<!-- Before -->
<script src="logic.js"></script>
<link rel="stylesheet" href="style.css">

<!-- After inlining -->
<script>
  /* full contents of logic.js */
</script>
<style>
  /* full contents of style.css */
</style>

External URLs (http://, https://, //) are left untouched.

Binary Assets

Images, fonts, and other binary files are base64-encoded and embedded as string literals in the Python wrapper. At runtime they are decoded and written to a temporary directory before mshta.exe is invoked.

Temp Directory Lifecycle

App launches â†’ Creates temp dir (prefix: hta_app_XXXX) â†’ Extracts all files to temp dir â†’ mshta.exe runs the HTA from temp dir â†’ User closes the HTA window â†’ atexit handler removes temp dir (shutil.rmtree)

The cleanup runs via Python's atexit module, so temp files are removed even if the process exits unexpectedly.

ğŸ”§ Troubleshooting

Layout Horizontal.Neon.TProgressbar not found

A ttk style registration issue in older builds. Fixed in current version â€” the progress bar style is initialized in App.__init__() using the built-in TProgressbar name. Make sure you are running the latest release.

No module named 'PyInstaller'

pip install pyinstaller

PermissionError / WinError 5

The output .exe is currently running. Close it before rebuilding.

mshta.exe not found on target machine

The generated EXE relies on mshta.exe which is part of Windows. Present on all standard installs. May be missing on Windows Server Core or disabled via Group Policy.

Icon is ignored / default Python icon

Either Pillow is not installed (for non-ICO formats) or the path is invalid. Check the Log for [WARN] messages about the icon.

UPX not found

Place upx.exe in the same folder as the converter, or add its directory to PATH. The status indicator shows âœ” when detected.

EXE crashes on target machine

  • Confirm mshta.exe is available on target
  • If your HTA uses ActiveX / COM â€” those must be registered on the target
  • Ensure no absolute paths in your HTA â€” all referenced files must be in the project folder to get bundled

ValueError: empty range in randrange (banner animation crash)

This was a bug in early builds where particles crashed on very narrow window widths. Fixed in current version with max() guards on the random range.

â“ FAQ

Does the generated EXE require Python on the target machine?

No. PyInstaller packages the Python runtime. Only mshta.exe (built into Windows) is needed.

Can I convert a multi-file HTA project?

Yes. Select the project folder. All supported assets are collected and bundled automatically.

Will CSS and JavaScript work correctly inside the EXE?

Yes. External references are inlined before bundling, so relative paths work as expected at runtime.

Can I use a PNG as the EXE icon?

Yes, with Pillow installed (pip install pillow). It is auto-converted to a multi-resolution .ico.

How much does UPX reduce file size?

Typically 40â€“70%. A 10 MB EXE may compress to 3â€“5 MB.

Can the converter itself run as an EXE (frozen by PyInstaller)?

Yes. When running frozen, it detects this via sys.frozen and searches PATH for a real Python interpreter (python, python3, then py) to run PyInstaller.

Where are settings stored?

%APPDATA%\hta2exe_gui\settings.json

My antivirus flags the generated EXE â€” is it a virus?

No. PyInstaller-generated executables are commonly false-positived by AV software, especially when UPX-compressed. Submit to your AV vendor as a false positive.
